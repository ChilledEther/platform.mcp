# Plan: Embed Templates

## Objective
Implement the template loading mechanism using Go's `embed` package to bundle assets into the binary.

## Context
- Depends on Plan 01-01 (Types).
- Templates live in `internal/templates`.
- Must support variable substitution.

## Tasks

- [ ] **Task 1: Template Structure & Dummy Templates**
  - Create `internal/templates` directory.
  - Add sample templates: `workflow.yaml.tmpl`, `docker-build.yaml.tmpl`.
  - Write test `internal/templates/loader_test.go` to verify templates can be read.

- [ ] **Task 2: Implement Loader (TDD)**
  - Create `internal/templates/loader.go`.
  - Use `//go:embed` to load templates.
  - Implement `Load(name string) (string, error)` function.
  - Write tests verifying:
    - Success reading existing template.
    - Error reading missing template.

- [ ] **Task 3: Template Rendering (TDD)**
  - Implement `Render(tmpl string, data any) (string, error)` using `text/template`.
  - Write tests with `Config` struct as data source.
  - Verify variable substitution works (e.g., `{{ .ProjectName }}`).

## Success Criteria
- [ ] Templates embedded in binary.
- [ ] Loader tests pass.
- [ ] Renderer tests pass with variable substitution.
